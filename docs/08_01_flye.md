# De novo genome assembly with `Flye`

---

## 🎯 Learning objectives

By the end of this practical, you should be able to:

- run Flye on Oxford Nanopore reads
- select the appropriate Flye read mode
- locate the main Flye output files
- assess basic assembly size, contiguity, coverage, and circularity

---

## Overview

`Flye` is a de novo assembler for long reads produced by Oxford Nanopore Technologies (ONT) and PacBio. Long reads can span repetitive regions and may allow a bacterial chromosome and plasmids to be assembled into a small number of contigs.

The workshop dataset contains ONT reads from four *Klebsiella pneumoniae* isolates. The reads were generated using R10.4.1 flow cells and basecalled with a Dorado super-accurate model with a minimum quality score of 10. We will therefore use Flye's `--nano-hq` mode.

{: .important }
> Flye requires long reads. Do not use the paired-end Illumina files as Flye input.

## 1. Install Flye

Move to the workshop directory and create the Conda environment:

```bash
cd ~/2026-bioinf-ToT-EPIC-Namibia

conda create -n flye \
  -c conda-forge \
  -c bioconda \
  flye seqkit \
  -y
```

Activate the environment and check the installed versions:

```bash
conda activate flye
flye --version
seqkit version
```

---

## 2. Select an ONT sample

The ONT reads downloaded in the dataset practical are located in:

```text
data/raw_data/kp_ont/
├── AR-1049-GC.fastq.gz
├── AR-1049-MV.fastq.gz
├── AR-1054.fastq.gz
└── AR-1054-GC.fastq.gz
```

List the available files:

```bash
cd ~/2026-bioinf-ToT-EPIC-Namibia
ls -lh data/raw_data/kp_ont/*.fastq.gz
```

Select one sample. This practical uses `AR-1049-MV` as an example:

```bash
SAMPLE=AR-1049-MV
READS="data/raw_data/kp_ont/${SAMPLE}.fastq.gz"
OUTDIR="analyses/flye/output/${SAMPLE}"

mkdir -p analyses/flye/output
ls -lh "$READS"
```

{: .note }
> To analyse another isolate, change only the value assigned to `SAMPLE`.

---

## 3. Inspect the reads

Summarize the input reads with SeqKit:

```bash
seqkit stats -a "$READS"
```

Record the following values:

- number of reads
- total number of bases
- mean read length
- read N50
- maximum read length

A rough coverage estimate is:

```text
coverage = total bases in reads / expected genome size
```

The expected total genome size for these isolates is approximately 5.7–5.8 Mb.

{: .task }
> Estimate the sequencing coverage of your sample. Is it sufficient for a bacterial long-read assembly?

---

## 4. Run Flye

Run Flye with eight CPU threads:

```bash
flye \
  --nano-hq "$READS" \
  --out-dir "$OUTDIR" \
  --threads 8
```

Flye writes progress information to the terminal and to:

```text
analyses/flye/output/<sample>/flye.log
```

Check whether the run completed successfully:

```bash
tail -n 25 "$OUTDIR/flye.log"
```

An interrupted run can usually be resumed with the same input and output directory:

```bash
flye \
  --nano-hq "$READS" \
  --out-dir "$OUTDIR" \
  --threads 8 \
  --resume
```

{: .important }
> Use `--resume` only when the input reads, read mode, and output directory are unchanged.

## 5. Inspect the assembly

The main Flye outputs are:

| File | Description |
|---|---|
| `assembly.fasta` | Final polished assembly used for downstream analyses |
| `assembly_info.txt` | Contig length, coverage, circularity, repeat status, and graph path |
| `assembly_graph.gfa` | Assembly graph in GFA format |
| `flye.log` | Flye log file |
| `params.json` | Parameters recorded for the run |

List the results and summarize the final assembly:

```bash
ls -lh "$OUTDIR"
seqkit stats -a "$OUTDIR/assembly.fasta"
```

Display the FASTA headers:

```bash
grep '^>' "$OUTDIR/assembly.fasta"
```

Inspect the contig information table:

```bash
column -t "$OUTDIR/assembly_info.txt" | less -S
```

Show the header and contigs marked as circular:

```bash
awk 'NR == 1 || $4 == "Y"' "$OUTDIR/assembly_info.txt" | column -t
```

{: .warning }
> A circularity flag is useful evidence, but it does not prove that a chromosome or plasmid is complete and correct.

---

## 6. Interpret the result

For a bacterial isolate, check:

- Is the total assembly size close to 5.7–5.8 Mb?
- Is there one chromosome-sized contig?
- Are smaller contigs present that may represent plasmids?
- Are any contigs marked as circular?
- Do the contigs have plausible and reasonably consistent coverage?

| Observation | Possible interpretation |
|---|---|
| One large circular contig | Potentially complete chromosome |
| Large contig plus small circular contigs | Chromosome and possible plasmids |
| Many short contigs | Low-quality reads, insufficient read length, contamination, or unresolved repeats |
| Assembly much larger than expected | Contamination, mixed culture, or duplicated sequence |
| Strongly different contig coverages | Plasmids, copy-number differences, or contamination |

{: .task }
> Record the total assembly size, number of contigs, N50, largest contig, and number of circular contigs. Does the result appear plausible for a *K. pneumoniae* isolate?

## Summary

The main commands and files are:

| Step | Command or file |
|---|---|
| Input reads | `data/raw_data/kp_ont/${SAMPLE}.fastq.gz` |
| Inspect reads | `seqkit stats -a "$READS"` |
| Assemble reads | `flye --nano-hq "$READS" --out-dir "$OUTDIR" --threads 8` |
| Final assembly | `analyses/flye/output/${SAMPLE}/assembly.fasta` |
| Contig information | `analyses/flye/output/${SAMPLE}/assembly_info.txt` |
